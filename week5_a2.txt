Question 7

"sort junk_file.txt > junk_file.txt" replaces everything in junk_file.txt with nothing. This occurs because > is destructive. In preparation, bash overwrites the file before the sort operation even begins, which means that bash has nothing to sort, and therefore emptiness replaces the lines in junk_file.txt.

You can sort the lines by doing the following:

cp junk_file.txt new_file.txt       # copy the junk file to a new file
sort new_file.txt > junk_file.txt   # sort that version, and reassign it to the original junk file
